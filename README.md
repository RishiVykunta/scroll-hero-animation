# ITZFIZZ: Scroll-Driven Hero Animation

A high-performance, minimalist, and seamless scroll-driven React Hero Section sequence designed to showcase fluid web interactions.

This project implements a dynamic multi-layered GSAP timeline handling bouncing entry staggers, background typography parallaxing, rapid motion-speed lines, and an animated custom SVG sports car that physically scrubs across the horizontal axis mapped 1:1 with the vertical scroll positioning.

The layout executes a deeply premium, sleek glassmorphic footprint anchored by individual CSS structural properties, preventing standard GSAP variable overlapping.

🔗 **Live Demo**: https://scroll-hero-animation-ten.vercel.app/
---

## 🚀 Tech Stack

*   **Next.js 15 (React 18)**: Core framework and server/client hydration.
*   **GSAP**: Primary physics animation engine targeting `opacity`, `transform` properties, and timeline manipulation.
*   **GSAP ScrollTrigger**: Links physical window scroll position smoothly into continuous timeline variables.
*   **Vanilla CSS 3**: Direct management of variables, dynamic glass filters (`backdrop-filter`), modern isolated CSS transitions (`rotate`, `scale`), and grid layout scaffolding.
*   **Tailwindcss**: Setup via Next.js for functional core baseline (mostly customized globally).
*   **Native Inline SVG**: All graphical assets are cleanly integrated natively leveraging inline React code execution.

## 🎨 Implementation Highlights

*   **Render Optimization**: GSAP Timelines are tightly wrapped in strict `gsap.context()` closures to guarantee automated cleanup operations on component un-mount, preventing memory leaking.
*   **React Strict Mode Compliant**: `useRef()` element arrays are explicitly loop-wiped natively within the render calculation sequence to protect animation staggers against duplicating artifacts generated by React executing double mounting procedures under the hood.
*   **BFCache Fixes**: External URLs are explicitly wrapped within `_blank` anchor architectures to circumvent the destructive Single Page App scroll-resets generated by backwards/forwards navigational timeline freezing.
*   **Modern Transform CSS**: GSAP layout calculations operate simultaneously but completely harmoniously against CSS global `:hover` styling functions through cleanly isolated modern layout definitions (`rotate`, `scale`) that bypass unified coordinate matrices (`transform`).
